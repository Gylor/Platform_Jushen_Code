## 一、项目总览

本项目是 **Aelos 人形机器人** 参加 **2026 CRAIC（跨平台具身智能创意挑战赛）** 的完整代码库，涵盖 **虚拟仿真** 和 **真实机器人控制** 两大环节。项目采用"仿真先行验证、真机落地执行"的双轨开发策略，确保代码在仿真环境中调试通过后再部署到真实机器人上运行。

### 项目顶层结构

## 二、模块一：craic_tongverse（虚拟仿真环节）

### 2.1 模块定位

在 **TongVerse 仿真平台**（基于 NVIDIA Isaac Sim）中运行的代码，用于在虚拟环境中模拟 Aelos 机器人完成比赛任务。仿真环境可以验证任务逻辑的正确性，避免在真机上反复调试造成硬件损耗。

### 2.2 目录结构

```

│   ├── start.py                  # 程序启动入口，定义任务关卡状态机
│   ├── voice.py                  # 语音/文本命令控制台（Unix Socket 通信）
│   └── robot_common/             # 公共方法库
│       ├── robot_host.py         # 机器人宿主类（位姿获取、动作执行、OCR识别）
│       ├── action_map.py         # 动作映射表（动作名 → JSON文件路径）
│       ├── coordinate_converter.py # 坐标系转换器（仿真坐标 ↔ 期望坐标）
│       └── voice_map.py          # 语音控制映射表
├── craic_scene/                  # 仿真场景资源

│   └── materials/                # 场景材质与模型

│   ├── GrabCube.json             # 抓取方块
│   └── ...（共40+个动作文件）
├── js_to_json/                   # JS动作文件 → JSON 转换工具
│   ├── convert.py                # 转换脚本（提取Lua代码并转为JSON）
```

### 2.4 任务流程

仿真环境中的比赛任务分为 4 个关卡：

1. **Task1 - 清洁桌面**：机器人移动到桌子旁，执行推抹布动作清洁桌面
2. **Task2 - 收纳方块**：机器人抓取黑色方块，搬运到二层书架上
3. **Task3 - 看板识别**：机器人弯腰用 OCR 识别看板文字，确定目标书籍和收纳箱颜色
4. **Task4 - 取书配送**：根据识别结果，从书架取对应书籍，放入对应颜色的收纳箱

### 3.4 技术栈

- **ROS Noetic**：机器人操作系统，负责话题通信
- **OpenCV**：图像处理与颜色识别
- **RapidOCR**：基于 ONNX Runtime 的中文 OCR 文字识别
- **Sherpa-ONNX**：离线流式语音识别
- **ar_track_alvar**：AR Tag 码检测与定位
- **leju**：乐聚机器人底层动作库（`base_action`、`music`）

### 4.4 动作执行流程

```
edu软件按键 → HKEY()读取键值 → req_actionLUA()查找动作名
    → _G[actname]()执行Lua函数
    → MOTOsetspeed()设置速度 → MOTOrigid16()初始化舵机
    → MOTOmove19()控制19个舵机角度 → MOTOwait()等待完成
    → act_completeLUA()标记完成
```

### 5.2 关键对应关系

| 仿真环节 | 真机环节 | 说明 |
|----------|----------|------|
| `start.py` 中的任务函数 | `robot_logic.py` 中的任务函数 | 相同的任务状态机逻辑，分别适配仿真和真机 |

| `motion_library/*.json` | `工程文件_EM动作函数下载/action/*.js` | 动作文件，JSON供仿真，JS供edu下载 |

### 5.3 开发工作流

1. **动作制作**：在 edu 图形化软件中编排动作 → 导出为 JS 文件
2. **动作转换**：运行 `convert.py` 将 JS 转为 JSON → 放入 `motion_library/`
3. **仿真验证**：在 TongVerse 中运行 `start.py`，调试任务逻辑
4. **真机部署**：将 JS 动作通过 edu 下载到机器人 → 运行 `robot_logic.py`
5. **语音扩展**：运行 `robot_voice.py` 启动语音控制

## 六、技术要点总结

### 6.1 传感器与感知

| 能力 | 仿真实现 | 真机实现 |
|------|----------|----------|
| 定位导航 | 仿真坐标 + `align_to_pose()` | AR Tag 码 + `align_to_tag()` |
| 颜色识别 | 仿真材质颜色 | OpenCV HSV 颜色空间 |
| 文字识别 | RapidOCR | RapidOCR |
| 语音控制 | 键盘输入模拟 | Sherpa-ONNX 流式ASR |
| 物体抓取 | 仿真物理引擎 | 舵机动作序列 |

### 6.2 动作系统

- **舵机数量**：19个（通过 `MOTOmove19` 控制）
- **动作类型**：空手行走（前进/后退/左移/右移/左转/右转）、抱箱行走、抓取/放下、上下楼梯、推抹布、倒地起身等
- **动作粒度**：支持快走/慢走/微调（如 `forward_q` 快走3步、`forward_s` 微调2cm）

### 6.3 通信方式

- **仿真**：Unix Socket（`/tmp/robot_control.sock`）用于语音控制台与主程序通信
